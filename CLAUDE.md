# Pääpäivä — Event Map

Helsinki event map webpage showing Pääpäivä locations on an interactive map with filtering and bilingual (FI/EN) support.

## Versioning

The CSS and JS entry points in `index.html` have a `?v=` query string for cache busting:

```html
<link rel="stylesheet" href="/src/styles/style.css?v=0.1.0" />
<script type="module" src="/src/main.js?v=0.1.0"></script>
```

**Bump both version strings in `index.html` whenever you deploy changes** so that browsers discard cached assets. Use semantic versioning — increment the patch number for small fixes, minor for new features.

## Commands

```bash
npm run dev           # Vite dev server at localhost:5173
npm run build         # Production build to dist/
npm run preview       # Preview production build
npm run prepare-data  # Geocode local locations.json → locations-resolved.json (fallback only)
```

## Data source — Google Sheets (real-time)

Location data is fetched live from Google Sheets every time the page loads. The sheet ID is in `src/config.js`. Geocoded coordinates are cached in the user's `localStorage` so Nominatim is only called for new or changed locations.

### One-time sheet setup (required)

The sheet must be published so the browser can fetch it without authentication:

1. Open the Google Sheet
2. **File → Share → Publish to web**
3. Select the correct tab → **Comma-separated values (.csv)** → **Publish**
4. Done — no URL needed in code, the sheet ID in `src/config.js` is sufficient

If the sheet is not published, the app shows an error: *"Sheet is not published."*

### Changing the sheet

Update `SHEET_ID` in `src/config.js`.

### Required sheet columns (first row = headers)

| Column | Required | Notes |
|---|---|---|
| `id` | Yes | URL-safe slug, e.g. `temppeliaukio` |
| `title` | Yes | Finnish title |
| `title_en` | No | English title |
| `address` | No | Full address — used if no coordinates |
| `lat` | No | Decimal latitude |
| `lng` | No | Decimal longitude |
| `time_start` | Yes | `HH:MM` format |
| `time_end` | Yes | `HH:MM` format (`00:00` = midnight) |
| `description` | No | Finnish description |
| `description_en` | No | English description |
| `tags` | No | Comma-separated, e.g. `music, culture` |

### Local JSON fallback

If not using Google Sheets, edit `src/data/locations.json` directly and run `npm run prepare-data`.

## Architecture

```
src/
  main.js           Orchestrator: wires map, filters, list, sidebar, language toggle
  map.js            Leaflet map, marker clustering, pan/highlight API
  filters.js        Filter state, DOM buttons, emits 'filtersChanged' CustomEvent
  locationList.js   Scrollable list below map
  i18n.js           Translations (fi/en), language state, emits 'langChanged' CustomEvent
  style.css         Layout, CSS custom properties for colours (edit :root to retheme)
  data/
    locations.json           Source data — edit this to add/change locations
    locations-resolved.json  Generated by prepare-data — do not edit by hand
scripts/
  prepare-data.js   Node script: reads locations.json, calls Nominatim, writes resolved JSON
```

## Data flow

`locations.json` → `npm run prepare-data` → `locations-resolved.json` → loaded at runtime by `main.js`

## Location data schema

```json
{
  "id": "unique-slug",
  "title": "Finnish title",
  "title_en": "English title",
  "address": "Street 1, Helsinki",  // null if coords-only
  "lat": 60.1234,                   // null if address-only; coords take precedence for map
  "lng": 24.9234,
  "time": { "start": "19:00", "end": "22:00" },
  "description": "Finnish description",
  "description_en": "English description",
  "tags": ["music", "culture"]
}
```

Three supported cases:
- **Address only** (`lat`/`lng` null): script forward-geocodes via Nominatim
- **Coords only** (`address` null): script reverse-geocodes via Nominatim  
- **Both**: coords used for map placement, address kept as-is

## Time filter logic

Filters: `all` | `current` | `upcoming` | `day` (10–17) | `evening` (17–21) | `lateNight` (21–)

- `current`: event's time window includes right now (uses `new Date()`)
- `upcoming`: event start time is in the future
- `day/evening/lateNight`: event overlaps with the window at all
- `"00:00"` end time is treated as 1440 minutes (midnight = end of day)

## Tag filter logic

Multi-select toggles. Empty selection = show all. OR logic within active tags.

## i18n

Language state lives in `src/i18n.js`. `toggleLang()` switches fi↔en and dispatches `langChanged` on `document`. `main.js`. listens and re-renders filter labels, location list, and sidebar.

Location text fields: `title` / `title_en`, `description` / `description_en`. If the `_en` field is absent, falls back to the Finnish field.

## Styling conventions

- All colours via CSS custom properties in `:root` (top of `style.css`) — change 4 variables to retheme
- No CSS framework, no preprocessor
- BEM-ish: `.location-item`, `.location-item__title`, `.location-item--active`

## Map notes

- Leaflet with OpenStreetMap tiles (no API key needed)
- `leaflet.markercluster` for clustering — its two CSS files must be imported in `map.js`
- All markers use `L.divIcon` (`.marker-dot`) to avoid Leaflet's default icon asset issues
- Custom cluster circles: 3 size tiers (28px / 36px / 48px) based on child count
- Map bounded to Greater Helsinki to prevent accidental panning

## Geocoding notes

- Never call Nominatim from the browser (rate limits, ToS)
- User-Agent header is required by Nominatim ToS — set in `prepare-data.js`
- `_geocodeSource`: `"forward"` | `"reverse"` | `"provided"` — used to skip re-geocoding on re-runs
- `_geocodeFailed: true` marks entries that failed; these are skipped on the map
